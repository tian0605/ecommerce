# 商品信息提取提示词 v1.0

## 任务
从商品标题和描述中提取关键信息，用于后续的Listing优化。

## 输入信息
- 商品标题：{original_title}
- 商品描述：{original_description}
- 热搜词候选（根据标题相似度匹配）：{hot_search_words}

## 提取要求

请从商品标题和描述中提取以下四个维度的信息：

### 1. attributes（商品属性）
从SKU和规格中提取：
- 尺寸/规格（如：30x20x15cm、10L、3層）
- 颜色（如：奶白色、灰色、紅色）
- 材质属性（如：布藝、HIPS、ABS、不鏽鋼）
- 数量/包装（如：一套、2件組）

格式：用「｜」分隔，如：30x20x15cm｜奶白色/奶黃色｜HIPS塑料

### 2. material（材质）
从描述中识别主要材质：
- 识别商品的主要构成材质
- 区分主材和辅料
- 如：HIPS塑料、ABS塑料、竹材+金屬

### 3. features（核心特点）
从标题和描述中提炼：
- 核心卖点（1-3个）
- 功能特性（如：可拆卸、折疊、防潮）
- 差异化特点（如：北歐風、卡通萌趣）

格式：用「、」分隔，如：5格分格設計、可拆卸分隔板、翻蓋設計

### 4. scenarios（适用场景）
根据商品特性推断：
- 适用的空间（如：桌面、梳妝台、浴室）
- 适用人群（如：兒童、學生、小資族）
- 收纳场景（如：首飾、髮飾、化妝品）

格式：用「、」分隔，如：桌面、梳妝台、兒童房

## 输出格式

请严格按照以下JSON格式输出，不要输出任何解释：

```json
{
  "attributes": "从SKU和规格提取的属性",
  "material": "主要材质",
  "features": "核心特点1、核心特点2、核心特点3",
  "scenarios": "场景1、场景2、场景3"
}
```

## 注意事项

1. **attributes vs material**：attributes是可视化的规格（用于标题），material是材质成分（用于描述）
2. **features vs scenarios**：features是产品本身的特点，scenarios是使用情境
3. **优先从SKU提取**：颜色、尺寸等优先从SKU信息中提取
4. **避免主观描述**：只提取客观信息，不要添加形容词

## 提取示例

### 示例1：发饰收纳盒
输入：
- 标题：发饰收纳盒桌面分格发圈发夹置物儿童带盖便携首饰盒
- 描述：材质：HIPS...颜色：奶白色、奶黄色...规格：大号...
- 热搜词：收納盒、首飾盒、生活用品

输出：
```json
{
  "attributes": "大號｜奶白色/奶黃色｜HIPS塑料",
  "material": "HIPS塑料（食品級安全材質）",
  "features": "5格分格設計、可拆卸分隔板、翻蓋便攜",
  "scenarios": "桌面、梳妝台、兒童房、首飾收納"
}
```

### 示例2：厨房沥水架
输入：
- 标题：304不锈钢沥水碗盘架厨房置物架
- 描述：双层、带接水盘...
- 热搜词：瀝水架、廚房、不鏽鋼

输出：
```json
{
  "attributes": "雙層｜不鏽鋼｜約30x20x40cm",
  "material": "304不鏽鋼",
  "features": "雙層大容量、瀝水通風、抗鏽耐用",
  "scenarios": "廚房洗碗槽、居家收納、餐廳備餐台"
}
```

**输出要求：只输出JSON格式，不要输出任何解释性文字。**
