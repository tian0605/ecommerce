- **1688 IP反爬限制：** 服务器IP访问1688被拒绝
- **原有方案瓶颈：** product-collector直接采集1688需要Browser Relay支持

### 1.2 新思路
利用**妙手ERP**作为中间层：
- 妙手ERP本身有1688插件，可以正常采集
- 服务器通过浏览器自动化操作妙手ERP的Web界面
- 绕过1688的IP反爬限制

### 1.3 目标
构建**全自动化**的商品采集 → 优化 → 上架 workflow

## 三、妙手ERP页面说明

### 3.1 页面1：产品采集

**URL：** `https://ms.aipaw.cn/collect/product`（示意）

**功能：** 发起1688商品采集

**操作流程：**
1. 进入「产品采集」页面
2. 输入1688商品链接（如 `https://detail.1688.com/offer/601234567.html`）
3. 点击「采集」按钮
4. 等待采集完成提示

**采集完成标志：**
- 显示"采集成功"提示
- 或页面自动跳转到公用采集箱

**注意事项：**
- 此页面仅用于发起采集
- 采集完成后数据自动进入公用采集箱
- 需要等待采集完成后再进行下一步

---

### 3.2 页面2：公用采集箱

**URL：** `https://ms.aipaw.cn/collect/list`（示意）

**功能：** 存储和管理已采集的商品

**操作流程：**
1. 进入「公用采集箱」页面
2. 在商品列表中找到目标商品
3. 点击「编辑」进入编辑页面
4. 在编辑页可以进行：查看详情、修改信息、回写数据

**编辑页面URL：** `https://ms.aipaw.cn/collect/edit/{product_id}`
