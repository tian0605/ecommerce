## 🔴 已解决

### 问题1：1688 IP反爬封锁
**日期：** 2026-03-20
**现象：** 直接爬取1688返回403 Forbidden
**原因：** 服务器IP被1688识别为爬虫
**教训：** 
- 不要直接在服务器上爬取1688
- 需要本地浏览器环境绕过
**解决方案：** 本地1688服务 + SSH隧道（见KNOWLEDGE.md）

### 问题2：Playwright线程冲突
**日期：** 2026-03-20
**现象：** 多个Playwright实例同时运行时崩溃
**原因：** Chromium的线程安全问题
**教训：** 浏览器实例需要串行使用
**解决方案：** `launch(threaded=False)`

### 问题3：货源ID提取失败
**日期：** 2026-03-20
**现象：** 货源ID显示为None
**原因：** 货源ID不在输入框，而是在链接里
**教训：** Vue动态渲染的数据需要从多个来源提取
**解决方案：** 从链接正则提取 `/offer/(\d+)\.html`

### 问题4：SKU数量硬编码误报
**日期：** 2026-03-25
**现象：** 心跳日志一直报"SKU数量应为3实际2"
**原因：** 测试商品实际只有2个SKU，脚本硬编码了期望值3
**教训：** 不要用硬编码值判断正确性，应该检查逻辑
**解决方案：** 移除错误的判断条件

### 问题5：status枚举值错误
**日期：** 2026-03-25
**现象：** listing-optimizer更新商品时报错 `invalid input value for enum product_status: "optimized"`
**原因：** 数据库product_status枚举只有 `collected, listed, published, delisted`
**教训：** 使用enum类型时要先确认可用值
**解决方案：** 直接UPDATE title和description字段，绕过status

## ⚠️ 避坑指南

1. **数据库enum类型**
   - 使用前先查询可用值
   - 不确定时用普通UPDATE而非依赖枚举

2. **LLM API调用**
   - 始终设置timeout
   - 失败时保留完整错误信息

3. **Playwright浏览器**
   - 不要同时启动多个实例
   - 使用完毕后务必close()

4. **SSH隧道**
   - 长时间不用会断开
   - 关键任务前检查隧道状态

5. **Cookies有效期**
   - 妙手Cookies通常24小时有效
   - 采集前检查修改时间
