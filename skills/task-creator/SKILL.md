# task-creator

## 技能：任务创建器

### 触发条件
用户提出新的任务或项目目标时激活此技能。

### 核心职责
接收用户设定的任务和成功标准，验证其清晰度，然后将任务写入工作流系统。

### 工作流程

```
用户提出任务 → 评估清晰度 → [不清晰?] → 追问 → [清晰?] → 写入队列
```

### 评估标准

**任务目标清晰度检查：**
1. 任务边界明确（做什么/不做什么）
2. 执行主体明确（谁来执行）
3. 资源/工具明确（用什么执行）
4. 时间/频率明确（何时执行）

**成功标准清晰度检查：**
1. 可验证（有明确的验证方法）
2. 可量化（有数字指标）
3. 有期限（有明确的完成时间）
4. 可观测（结果可被观察到）

### 不清晰时的追问模板

**任务目标不清晰时：**
```
我需要确认几个问题：

1. 这个任务具体要做什么？
   - 需要执行哪些步骤？
   - 有什么是明确不包括的？

2. 谁来执行这个任务？
   - 我（AI agent）自动执行？
   - 还是需要你手动操作？

3. 使用什么工具/模块来执行？
   - 有特定的脚本或技能可以使用吗？

4. 什么时候需要完成？
   - 是一次性任务还是定期任务？
```

**成功标准不清晰时：**
```
我需要确认如何判断任务完成了：

1. 怎么验证成功了？
   - 需要查看什么具体指标？
   - 需要检查什么文件/数据库/输出？

2. 有具体的数字标准吗？
   - 比如：成功率 > 90%
   - 比如：完成时间 < 5 分钟

3. 完成后需要什么证据？
   - 飞书通知？
   - 文档记录？
   - 数据库更新？
```

### 写入格式

**dev-task-queue.md:**
```markdown
## 📋 任务：[任务名称]

**创建时间：** YYYY-MM-DD HH:MM
**创建者：** 用户/AI

### 任务目标
[清晰的任务描述]

### 成功标准
| 标准 | 验证方法 | 状态 |
|------|----------|------|
| [标准1] | [验证方法] | ⬜ |
| [标准2] | [验证方法] | ⬜ |

### 执行步骤
1. [步骤1]
2. [步骤2]
```

**task_state.json:**
```json
{
  "task_name": "任务名称",
  "created_at": "ISO时间",
  "success_criteria": [...],
  "status": "pending",
  "current_step": 0
}
```

### 使用示例

**用户输入：**
> 帮我每天自动更新热搜词库

**AI 评估：**
- 任务边界：清晰（更新热搜词库）
- 执行主体：不明确（AI自动还是手动触发？）
- 工具：需要指定（爬虫脚本？数据源？）
- 时间：需要明确（每天几点？触发方式？）

**追问：**
```
我来帮你创建热搜词库更新任务，但需要确认几个问题：

1. **数据源**：从哪个平台/文件获取热搜词？
   - 是之前导入的 xlsx 文件吗？
   - 还是有新的数据源？

2. **执行方式**：
   - 由心跳自动触发（每10分钟检查）？
   - 还是每天定时执行一次？

3. **成功标准**：
   - 怎么算"更新成功"？
   - 需要验证什么？（如：数据库记录数 > 400）
```

**用户回复后 → 写入队列**

## 查询任务清单

### 命令
```bash
python3 /root/.openclaw/workspace-e-commerce/skills/task-creator/task_creator.py list
```

### 输出内容

**【当前执行状态】**
- 任务名称
- 创建时间
- 状态（pending/in_progress/completed）
- 完成度（当前步骤/总步骤数）
- 成功标准列表

**【任务队列】**
- P0 问题清单
- 待执行任务
- 进行中任务
- 已完成任务

**【最近执行日志】**
- task_executor.log 最后10行

**【统计】**
- P0 问题数量
- 待执行数量
- 已完成数量

### 使用场景

**场景1：检查当前有什么任务**
```
用户: 现在有哪些任务在执行？
AI: (调用 list_tasks)
```

**场景2：检查某个任务的成功标准**
```
用户: listing-optimizer 的成功标准是什么？
AI: (调用 list_tasks 查看成功标准列表)
```

```
用户: 任务为什么没完成？
AI: (调用 list_tasks 查看执行状态和日志)
```
