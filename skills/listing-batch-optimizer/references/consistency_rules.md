# 一致性检查规则

## 检查项目

### 1. 关键词覆盖率
- 提取标题中的关键词（2字以上）
- 检查前3-5个关键词是否在描述前300字出现
- 通过率：100%（每个关键词都应出现）

### 1.1 头部补强机制
- 若模型生成的描述前200字未覆盖标题前4个关键词，脚本会先执行一次“关键词头部补强”
- 补强方式：在描述最前面插入一行 `【重點整理】...`，显式补齐缺失关键词
- 补强完成后，再进入最终一致性校验
- 该机制只补关键词覆盖，不绕过长度、禁用词、格式等其他检查

### 2. 格式一致性
| 检查项 | 标题有 | 描述必须有 |
|--------|--------|------------|
| ✨符号 | ✨ | 描述前100字内必须有✨ |
| 【】符号 | 【 | 描述前100字内必须有【 |
| 括号() | ( | 描述中应体现 |

### 3. 品类词位置
- 标题中的核心品类词（如"露營燈架"、"收納盒"）
- 必须在描述前150字出现
- 不能在描述后半部分才出现

### 3.1 维护建议
- 若某一类商品经常在“描述前200字缺少标题关键词”处失败，优先检查是否是关键词过长、首段过长，或模型把关键词写到了第二段以后
- 这类问题优先通过“头部补强”兜底，而不是直接放宽校验阈值
- 若补强后仍失败，再考虑调整关键词提取或提示词结构

### 4. 数值一致性
- 标题中的尺寸、容量、数量
- 描述中必须体现（不能矛盾）
- 例如：标题写"12種顏色"，描述不能写"8種"

### 5. SKU一致性
- 标题或描述中提到的规格数量
- 应与实际SKU数量匹配
- 允许范围：实际数量 ± 2

## 评分标准

| 分数 | 标准 |
|------|------|
| 90-100 | 完全一致，无问题 |
| 70-89 | 小问题（1-2个关键词缺失） |
| 50-69 | 中等问题（格式不一致） |
| <50 | 严重不一致（品类词缺失） |

## 修复优先级

1. **P0**：品类词缺失 - 立即修复
2. **P1**：关键词缺失 > 3个 - 今日修复
3. **P2**：关键词缺失 1-3个 - 本周修复
4. **P3**：格式不一致 - 本周修复
