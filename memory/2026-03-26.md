### 2. 日志机制问题发现与修复

**问题**: TC-FLOW-001 状态是 processing 但 main_logs 无记录

**根本原因**: 
- log.set_task(task_name) 和 log.finish("running") 未在 subprocess 执行前调用
- mark_start() 只更新数据库，不写日志
- subprocess 超时被杀时，log.finish() 从未执行

**修复**:
```python
# prod_task_cron.py
tm.mark_start(task_name)
log.finish("running")  # 预先写入开始日志
# 然后才执行 subprocess
```

### 3. prod_task_cron 增强监控

**新功能** (commit 8e3a1ca):
1. **Popen实时输出**: 使用 subprocess.Popen 替代 subprocess.run，实时打印stdout
2. **卡死检测**: 10分钟无日志判定为卡死
3. **自动处理卡死任务**: 杀掉进程 + 重置状态为 error_fix_pending
4. **运行中任务日志**: 插入"任务正常运行中，等待下一次检查"

**关键代码**:
```python
STUCK_TIMEOUT_MINUTES = 10

def is_task_stuck(task_name):
    last_time = get_task_last_log_time(task_name)
    return datetime.now() - last_time > timedelta(minutes=STUCK_TIMEOUT_MINUTES)
```
