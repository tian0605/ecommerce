## 18:xx-19:26 关键Bug修复

### 问题：retry_count >= 3 但父任务未设置 error_fix_pending

**现象：**
- TC-FLOW-001 重试次数不断增加（7次+）
- 日志显示："重试X次，先进行根因分析..."
- 但父任务一直是 `processing` 状态
- 子任务已创建但父任务继续重试

**根因：**
`prod_task_cron.py` 第270行左右，当 `retry_count >= 3` 时：
1. 调用根因分析
2. 创建子任务
3. 记录日志 `log.finish("failed")`
4. **但没有设置 `exec_state = 'error_fix_pending'`**

导致下次循环时，父任务仍被标记为 `processing`，继续重试。

**修复：**
在 `create_fix_subtasks` 后添加：
```python
# 重要：设置error_fix_pending停止重试循环
tm.update_task(task_name, exec_state='error_fix_pending')
```

**修复位置：** `/root/.openclaw/workspace-e-commerce/scripts/prod_task_cron.py` 第271行

### 关键经验
1. **创建子任务后必须立即设置父任务为 `error_fix_pending`**，否则重试循环不会停止
